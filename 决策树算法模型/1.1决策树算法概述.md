## 1. 决策树核心思想

决策树是一种**树形结构**的监督学习算法，模拟人脑“分而治之”的决策逻辑。  
- 每个**内部节点**表示一个特征上的测试（例如“湿度是否高？”）  
- 每个**分支**对应一个测试结果  
- 每个**叶子节点**给出一个预测值（分类任务为多数类，回归任务为均值）

学习过程是**递归分裂**：从根节点（全部数据集）开始，每次选择一个特征和分裂点，将当前节点数据划分到子节点中，使得子节点的“纯度”尽可能高。重复直至满足停止条件。

---

## 2. 树的结构与构建流程

1. **选择最优分裂特征和分裂点**  
   遍历所有特征，对每个特征尝试所有可能的分裂点，计算分裂后的纯度提升（或误差下降），选择提升最大的特征和分裂点。

2. **划分数据集**  
   根据选定的分裂规则将当前节点数据分配到两个（或多个）子节点中。

3. **递归构建**  
   对每个子节点重复步骤1~2，直到达到停止条件（如节点样本数过少、纯度已为0、达到最大深度等）。

4. **停止并生成叶子节点**  
   若节点无法再分裂，则设为叶子节点，输出该节点中样本的多数类（分类）或均值（回归）。

---

## 3. 分裂准则详解

* 分裂准则是用来量化“纯度”或“误差”的指标，指导我们选择最优分裂。
* 分裂准则不是“算法”，决策树的算法不是一个具体的可抽象的函数表达式，而是一套完整构建流程（包括如何处理特征、如何停止、如何剪枝等）。
### 3.1 分类树

#### 3.1.1 基尼系数（Gini Index）
- **定义**：  
  $$
  Gini = 1 - \sum_{k=1}^{K} p_k^2
  $$ 
  其中 $p_k$ 是第 $k$ 类样本在节点中的比例。
- **性质**：  
  - 节点越纯，基尼系数越小（纯节点为0）；  
  - 两类均匀分布时基尼最大（如二分类各半时为0.5）。
- **分裂选择**：计算分裂后的加权平均基尼，选择**基尼减少量**最大的特征/分裂点。  
  $$
  \Delta Gini = Gini_{parent} - \sum \frac{n_{child}}{n_{parent}} Gini_{child}
  $$
- **应用**：CART 分类树默认使用基尼系数。

#### 3.1.2 信息增益（Information Gain）
- **信息熵**：  
  $$
  Entropy = -\sum_{k=1}^{K} p_k \log_2 p_k
  $$  
  熵衡量不确定性，纯节点熵为0，最大熵为 $\log_2 K$（二分类最大为1）。
- **信息增益**：  
  $$
  IG = Entropy_{parent} - \sum \frac{n_{child}}{n_{parent}} Entropy_{child}
  $$
- **分裂选择**：选择信息增益最大的特征/分裂点。
- **缺点**：倾向于选择取值较多的特征（如ID列）。
- **应用**：ID3 算法使用信息增益。

#### 3.1.3 增益率（Gain Ratio）
- **动机**：修正信息增益对多取值特征的偏好。
- **定义**：  
  $$
  GainRatio = \frac{IG}{SplitInfo}, \quad SplitInfo = -\sum \frac{n_{child}}{n_{parent}} \log_2 \frac{n_{child}}{n_{parent}}
  $$  
  SplitInfo 反映了分裂本身产生的分支数及分布均匀程度，分支越多、越均匀，SplitInfo 越大，起到惩罚作用。
- **应用**：C4.5 算法使用增益率。

> **对比**：基尼系数和信息增益在大多数情况下选择效果接近，基尼计算更快；增益率修正了信息增益的偏向，但可能过度惩罚导致偏向分支少但增益不大的特征。

### 3.2 回归树

#### 3.2.1 均方误差（MSE）
- **定义**：节点内样本真实值 $y_i$ 与节点预测值 $\bar{y}$ 的平方差的平均值：  
  $$
  MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \bar{y})^2
  $$  
  其中 $\bar{y}$ 为该节点所有样本的均值。
- **分裂选择**：遍历特征和分裂点，使分裂后**子节点加权 MSE 之和最小**，等价于最大化 MSE 减少量。
- **应用**：CART 回归树默认使用 MSE（在 sklearn 中为 `squared_error`）。

## 4. 用“打网球”例子完整拆解构建过程

我们使用经典的“Play Tennis”数据集（14条记录），目标是根据天气决定是否去打网球。

### 4.1 原始数据

| Outlook | Temperature | Humidity | Windy | Play |
|---------|-------------|----------|-------|------|
| Sunny   | Hot         | High     | False | No   |
| Sunny   | Hot         | High     | True  | No   |
| Overcast| Hot         | High     | False | Yes  |
| Rainy   | Mild        | High     | False | Yes  |
| Rainy   | Cool        | Normal   | False | Yes  |
| Rainy   | Cool        | Normal   | True  | No   |
| Overcast| Cool        | Normal   | True  | Yes  |
| Sunny   | Mild        | High     | False | No   |
| Sunny   | Cool        | Normal   | False | Yes  |
| Rainy   | Mild        | Normal   | False | Yes  |
| Sunny   | Mild        | Normal   | True  | Yes  |
| Overcast| Mild        | High     | True  | Yes  |
| Overcast| Hot         | Normal   | False | Yes  |
| Rainy   | Mild        | High     | True  | No   |

### 4.2 第一次分裂：选择根节点

我们使用**基尼系数**进行演示。根节点有 9 个 Yes、5 个 No，基尼 = \(1 - (9/14)^2 - (5/14)^2 = 0.459\)。

#### 按 Outlook 分裂：
- Sunny (2Y,3N): 基尼 = 0.48  
- Overcast (4Y,0N): 基尼 = 0  
- Rainy (3Y,2N): 基尼 = 0.48  
加权平均基尼 = $(5/14)*0.48 + (4/14)*0 + (5/14)*0.48 = 0.343$  
基尼减少量 = 0.459 - 0.343 = 0.116

#### 按 Humidity 分裂：
- High (3Y,4N): 基尼 = 0.490  
- Normal (6Y,1N): 基尼 = 0.245  
加权平均基尼 = 0.3675  
基尼减少量 = 0.459 - 0.3675 = 0.0915

类似可算出 Temperature 和 Windy 的基尼减少量均小于 Outlook，因此选择 **Outlook** 作为根节点。

### 4.3 继续分裂子节点

**Overcast** 分支已全 Yes，成为叶子节点，预测 Yes。

**Sunny** 分支（5条数据）：  
- 再按 Humidity 分裂：  
  - High (0Y,3N): 基尼 0 → 纯 No  
  - Normal (2Y,0N): 基尼 0 → 纯 Yes  
  因此 Sunny 节点下选择 Humidity 分裂，形成两个叶子。

**Rainy** 分支（5条数据）：  
- 按 Windy 分裂：  
  - False (3Y,0N): 纯 Yes  
  - True (0Y,2N): 纯 No  
  因此 Rainy 节点下选择 Windy 分裂。

### 4.4 最终树结构

```
                  Outlook
                /    |    \
           Sunny Overcast Rainy
          /   \     |     /   \
     Humidity   (Yes)  Windy
      /    \          /    \
   High  Normal    False  True
    |      |         |      |
   No     Yes       Yes     No
```

### 4.5 预测过程

- 新样本：Outlook=Sunny, Humidity=High → 走到 Sunny → Humidity=High → 预测 No。
- 新样本：Outlook=Rainy, Windy=False → 走到 Rainy → Windy=False → 预测 Yes。

## 5. 过拟合与剪枝

决策树容易对训练数据过度学习，导致泛化能力下降（过拟合）。剪枝是控制过拟合的核心手段。

### 5.1 预剪枝（Pre-pruning）
在树生长过程中提前停止分裂，常用参数：
- 限制最大深度 (`max_depth`)
- 要求节点最小样本数 (`min_samples_split`)
- 要求叶子节点最小样本数 (`min_samples_leaf`)
- 分裂所需的最小不纯度下降 (`min_impurity_decrease`)

**优点**：计算快，简单有效；**缺点**：可能欠拟合。

### 5.2 后剪枝（Post-pruning）
先让树充分生长，再自底向上评估是否将子树替换为叶子节点（通常基于验证集误差）。sklearn 中通过 `ccp_alpha` 实现**代价复杂度剪枝**：
- 对于每个子树，定义代价复杂度 \( R_\alpha(T) = R(T) + \alpha |T| \)  
- 选择使代价复杂度最小的子树，\(\alpha\) 越大，树越精简。

## 6. sklearn 中的决策树实现

sklearn 的 `DecisionTreeClassifier` 和 `DecisionTreeRegressor` 基于 CART 算法改进，支持连续特征、预剪枝、代价复杂度剪枝等。

### 6.1 分类树

```python
from sklearn.tree import DecisionTreeClassifier

clf = DecisionTreeClassifier(
    criterion='gini',        # 分裂准则: 'gini' (CART) 或 'entropy' (ID3风格)
    max_depth=5,             # 最大深度
    min_samples_split=2,     # 内部节点再划分所需最小样本数
    min_samples_leaf=1,      # 叶子节点最少样本数
    max_features=None,       # 寻找最佳分裂时考虑的特征数量
    ccp_alpha=0.0,           # 代价复杂度剪枝参数（非负）
    random_state=42
)
clf.fit(X_train, y_train)
```

### 6.2 回归树

```python
from sklearn.tree import DecisionTreeRegressor

reg = DecisionTreeRegressor(
    criterion='squared_error', # 分裂准则: 'squared_error' (MSE), 'absolute_error' (MAE) 等
    max_depth=5,
    min_samples_split=2,
    min_samples_leaf=1,
    ccp_alpha=0.0
)
reg.fit(X_train, y_train)
```

### 6.3 重要参数解读

| 参数 | 作用 | 调参建议 |
|------|------|----------|
| `criterion` | 分裂准则 | 分类用 'gini' 或 'entropy'；回归用 'squared_error' |
| `max_depth` | 最大深度 | 限制深度是控制过拟合最直接的方式，常用 3~10 |
| `min_samples_split` | 节点最小样本数 | 增大可防止过度分裂，常用 5~20 |
| `min_samples_leaf` | 叶子最小样本数 | 与 min_samples_split 类似，通常设为 1~10 |
| `ccp_alpha` | 代价复杂度剪枝 | 非零值会剪掉一些子树，值越大剪枝越强，常用交叉验证选择 |
| `max_features` | 每次分裂考虑的特征数 | 用于引入随机性（如随机森林），单棵树通常设为 None |

### 6.4 可视化

```python
from sklearn.tree import plot_tree
import matplotlib.pyplot as plt

plt.figure(figsize=(12,8))
plot_tree(clf, filled=True, feature_names=feature_names, class_names=class_names)
plt.show()
```

## 7. 对训练样本的要求与预处理

决策树对数据的要求相比线性回归宽松得多，但仍有几点需注意：

| 方面 | 要求 / 建议 |
|------|------------|
| **数值特征** | 直接使用，无需标准化或归一化。 |
| **类别特征** | sklearn 决策树不支持字符串，需编码为整数（OrdinalEncoder 或 OneHotEncoder）。OrdinalEncoder 会引入顺序假设，OneHotEncoder 会扩展特征维度，需根据业务场景选择。 |
| **缺失值** | 训练数据中不能有 NaN，需提前填充（如 SimpleImputer 用均值/众数）。 |
| **样本量** | 小样本（<100）时需严格限制深度，防止过拟合；大样本（>10万）时需注意计算效率，可设置 `max_features` 或降低精度。 |
| **异常值** | 鲁棒性较强，极端值仅影响所在分支，无需特殊处理。 |

## 8. 决策树优缺点总结

| 优点 | 缺点 |
|------|------|
| 模型可解释性强，可视化后直观易懂 | 容易过拟合，需剪枝或集成 |
| 无需特征缩放，对非线性关系天然适应 | 对数据微小变化敏感，方差较大 |
| 能处理混合类型特征（数值+类别） | 可能偏向取值较多的特征（信息增益情况下） |
| 自动进行特征选择 | 在特征强相关时表现不稳定 |

## 9. 附录：算法与分裂准则对应关系

| 算法 | 分裂准则 | 特征处理 | 剪枝 | sklearn 对应 |
|------|----------|----------|------|--------------|
| **ID3** | 信息增益 | 仅离散特征 | 无 | `criterion='entropy'`（但支持连续特征，有预剪枝） |
| **C4.5** | 增益率 | 离散+连续，支持缺失值 | 后剪枝 | 无直接对应，可通过 `ccp_alpha` 近似后剪枝 |
| **CART（分类）** | 基尼系数 | 离散+连续，不支持缺失值 | 代价复杂度剪枝 | `criterion='gini'`，`ccp_alpha` |
| **CART（回归）** | 均方误差 (MSE) | 离散+连续，不支持缺失值 | 代价复杂度剪枝 | `criterion='squared_error'` |

> 注意：sklearn 的实现是 CART 的增强版，统一使用二叉树分裂（即使是离散特征也二分），支持连续特征和预剪枝，因此实际使用时不必严格区分算法，只需根据任务选择合适的分裂准则即可。
